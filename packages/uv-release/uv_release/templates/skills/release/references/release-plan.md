# The Release Plan

The release plan is a self-contained JSON document passed as the `plan` input to the workflow. It contains:

- A list of jobs (validate, build, release, publish, bump) with their commands
- A runner matrix (one entry per unique runner configuration)
- Python version for CI builds
- Publish environment name for trusted publishing
- A skip list (job names to skip)
- Reuse configuration (`reuse_run` for artifact reuse, `reuse_release` to skip tagging)

CI reads everything from this plan. It does not inspect git history or compute anything at runtime. The release and publish jobs include download commands that fetch build artifacts from the current or a prior CI run.

The plan is generated by `uvr release` and can be previewed with `uvr release --json`. See `cmd-release.md` for details.
