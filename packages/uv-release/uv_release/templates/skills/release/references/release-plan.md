# The Release Plan

The release plan is a self-contained JSON document passed as the `plan` input to the workflow. It contains:

- Version numbers and build commands for each changed package
- A runner matrix (one entry per unique runner configuration)
- A release matrix (one entry per changed package)
- A skip list (job names to skip)
- Reuse configuration (prior run ID for artifact reuse)

CI reads everything from this plan — it does not inspect git history or compute anything at runtime.

The plan is generated by `uvr release` and can be previewed with `uvr release --json`. See `cmd-release.md` for details.
