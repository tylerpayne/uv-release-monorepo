# Record Architecture Decisions

* Status: accepted
* Date: 2026-03-25

## Context and Problem Statement

We need to record the architectural decisions made on this project so that future contributors can understand why things are the way they are.

## Decision Outcome

Chosen option: Use Architecture Decision Records (ADRs) in [MADR](https://adr.github.io/madr/) format, managed by [pyadr](https://pypi.org/project/pyadr/).

- Workspace-level decisions live in `docs/adr/`
- Package-level decisions live in `packages/<pkg>/docs/adr/`
